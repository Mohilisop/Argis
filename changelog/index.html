    Changelog¶

    0.9.0 — Engine Overhaul, Encoding Safety & Data Integrity¶

    Engine Fixes (scanning accuracy)¶

    • Fixed title detection bug — title search window widened from 5,000 → 50,000 bytes. Many sites (GitHub, etc.) have their <title> tag beyond 5 KB, causing every valid account to return NOT_FOUND. This was the root cause of missing accounts.
    • Confidence scoring now uses 30,000 bytes of page content (was 8,000) for more accurate analysis
    • Description extraction widened from 5,000 → 50,000 bytes
    • Non-encodable characters are now stripped from titles and descriptions, preventing crashes on terminals that don't support emoji/exotic Unicode

    Output / Encoding Safety¶

    • Logo converted to pure ASCII — the previous Unicode block-art (█) crashed on Windows cp1252 terminals before any scan could run
    • Status bars (_status_bar, print_completion) auto-detect whether the terminal supports the FULL BLOCK character and fall back to # if not
    • Windows console now enables Virtual Terminal Processing + UTF-8 mode, letting Rich use the modern rendering path instead of the legacy cp1252 codec path
    • Logo subtitle changed from ⚡ SIGINT COLLECTOR to > SIGINT COLLECTOR (ASCII-safe)

    Data Integrity¶

    • Removed duplicate "Throne" entry from sites.json (was listed twice under content)
    • Fixed hardcoded path in import-sites command — now resolves from package location instead of CWD
    • Updated all stale version strings (0.8.0 → 0.9.0) across docs, home screen, and package metadata
    • Updated mkdocs.yml description to reflect 500+ platforms (was "133+")
    • Expanded .gitignore with generated output patterns (*.html, health*.md, dossier*.html, etc.)

    Cleanup¶

    • media_runtime.py stub restored after accidental removal
    • test_profile_media_apis.py removed (tested unimplemented functions from stub)
    • 60+ core tests passing, full suite verified

    0.8.0 — Full Export Suite + AI Analysis¶

    • New export formats: TXT (-T), NDJSON (-J ndjson), XMind 8 (-X), GraphML (-G), Neo4j Cypher (--neo4j), PDF (-P)
    • AI-powered risk analysis via --ai (OpenAI GPT-4o / Anthropic Claude)
    • FORMATTERS registry extended with new formatters
    • 12 new tests covering all export formats + 3 AI analysis tests
    • 166 tests, all passing

    0.7.0 — Intelligence Ecosystem¶

    • argis breach — check emails against known breaches (HIBP-style)
    • argis mentions — web-mention search via Google dorks
    • argis geo — geographic region inference from profile signals
    • argis me — unified self-assessment: scan + breach + mentions + geo + impersonation in one command
    • argis mcp — run Argis as a Model Context Protocol server
    • Confidence scoring (0-100) on every FOUND hit + --min-confidence filter
    • Shared extract_utils.py — email, URL, and title extraction refactored out of core.py and correlate.py
    • Richer challenge/block detection with expanded _CHALLENGE_MARKERS
    • MCP server as optional dependency (pip install "argis[mcp]")
    • 151 tests, all passing

    0.6.0 — Dossier v2¶

    • Full HTML dossier with interactive knowledge graph (vis-network)
    • Dossier PDF export (weasyprint + playwright fallback)
    • Screenshot capture + terminal rendering
    • Rich scan results table with terminal screenshots

    • Weekly GitHub Action for automatic doctor runs
    • Duplicate rule name detection in sites.json
