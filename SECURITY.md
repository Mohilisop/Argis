# Security Policy

Argis is designed exclusively for **defensive / self-OSINT** — scanning your own usernames, monitoring your own footprint, and protecting against impersonation. It must never be used to target individuals without their explicit consent.

## Supported Versions

| Version | Supported |
|---------|-----------|
| >= 0.9.x | ✅ |
| < 0.9.0  | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in Argis, please report it privately by emailing **security@example.com**.

Do not open a public GitHub issue for security vulnerabilities.

We aim to acknowledge receipt within 48 hours and provide an initial assessment within 5 business days.

## Scope

- Unauthorized data access or leakage through Argis
- Remote code execution or privilege escalation
- Abuse of Argis for harassment or doxxing
- Circumvention of rate limits or anti-bot protections built into the tool

## Out of Scope

- Social engineering of Argis users
- Attacks on the underlying OS or Python runtime
- Third-party platforms that Argis scans (report those to the platform directly)

## Disclosure Policy

We practice coordinated disclosure. Once a fix is released, we will credit the reporter (with permission) in the release notes.
